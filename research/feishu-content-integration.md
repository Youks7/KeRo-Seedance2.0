# 飞书资料对 Seedance Skill 的内容整合研究

研究日期：2026-07-13
研究对象：主飞书页《Seedance 2.5实战教程》、其两个子资料库、本地完整抓取物，以及当前 `seedance-video-production` Skill。
结论性质：内容审计与整合建议；**本报告没有修改 `SKILL.md`、`references/` 或 `evals/`**。

## 结论先行

这批飞书资料有价值，但不适合整体并入 Skill。当前 Skill 已经正确做了“短成品优先、素材职责隔离、可见因果、最小修改、平台事实单独核验”的减法重构。飞书资料真正能带来增量的，不是更多形容词、长提示词或电影名，而是四种尚未充分产品化的控制结构：

1. **参考视频的三段式作用域合同**：明确“参考什么、忽略什么、哪些目标素材优先”，防止动作参考污染身份、服装、场景和光色。
2. **转场的双端状态合同**：不只维护连续性，还要为匹配剪辑分别设计 A 镜头出点和 B 镜头入点，并选择动作、形状、颜色、构图或声音桥作为匹配维度。
3. **风险优先的项目验证顺序**：批量生成前先验证环境、身份、关键叙事/特效和结尾接力中的最高风险镜头；天气或复杂效果先单独测试，再扩散到多个镜头。
4. **按叙事意图检索镜头语言**：把运镜、构图、蒙太奇和电影案例压缩成“意图 → 可见手段 → 使用条件 → 失败风险”，而不是把术语百科和 23 部影片全文塞进上下文。

建议只向核心 `SKILL.md` 增加 2–3 条跨任务决策规则；其余内容放入按需参考；再新增 6 个针对性的 eval。不要把飞书原文、整套电影案例、固定平台参数或通用负面词表镜像进 Skill。

审计结束时，工作树中已经出现三个尚未由 `SKILL.md` 路由的参考草稿：`references/film-grammar.md`、`references/storyboard-and-production.md`、`references/visual-direction.md`。它们已经覆盖电影语法索引、场景功能、核心测试镜头、双端转场、视觉圣经和天气环境反应。因而当前最重要的不是再建三个同义文件，而是：核对这些草稿、从 `SKILL.md` 正确路由它们、补上参考视频“采用/忽略/覆盖”合同，并用 eval 固化行为。

## 2. 应进入核心 SKILL 的规则

核心规则必须同时满足三个条件：跨任务高频、能直接减少生成失败、用很短的表述即可执行。按这个门槛，建议只增加以下三项。

### 2.1 参考视频必须声明“采用 / 忽略 / 覆盖”

**建议规则**：每个 `@视频N` 至少声明一个采用层（动作节奏、表演微动作、运镜或剪辑节奏）、一个明确忽略层（身份、服装、场景、光色等），以及冲突时由哪一个图片或文本约束覆盖；只写“参考这个视频”不合格。

**为什么是核心增量**：当前 `reference-workflows.md` 已要求拆开动作与运镜控制层，但没有把冲突处理固化成完整合同。飞书教程明确给出“参考什么、忽略什么、迁移到什么角色”的结构，并指出文本与视频冲突时要明确优先级。〔`feishu-main-text.txt`，§9.3“表演迁移提示词写法”，行 533–543；§9.5“参考视频与文本提示词的配合”，行 554–560；对应 `feishu-corpus/main/Seedance 2.5实战教程.txt` 同行〕

**与现有 Skill 的关系**：这是对 `references/reference-workflows.md` §“动作与运镜参考”的补全，不推翻 `SKILL.md` §“多素材制作”中的唯一主职责原则。

### 2.2 多镜头先判断“连续接力”还是“有意剪辑”

**建议规则**：相邻镜头先选一种关系：

- 连续接力：复述人物、道具、摄影、环境和声音的出点状态；
- 有意剪辑：指定动作、形状、颜色、构图位置或声音中的一个匹配维度，并分别写出 A 镜头结束画面与 B 镜头开始画面。

不要用“自然转场”“无缝衔接”代替可见的两端状态。

**为什么是核心增量**：当前 Skill 对跨段连续性处理较强，但对“主动的不连续”只有很弱的覆盖。飞书主教程给出了转头动作的 A/B 双端设计，并把匹配剪辑、空镜和声音桥区分开；《2001太空漫游》案例则展示了形状匹配如何承载跨时代跳切。〔`feishu-main-text.txt`，§4.5“转场与镜头衔接”，行 300–314；§12.4“转场与匹配剪辑”，行 700–710；`feishu-corpus/movie-cases/2001太空漫游 分镜详解.txt`，“分镜 2：骨头抛天——影史最伟大的匹配剪辑”，行 17–27；JSON record `3C75049978D6080642A4099D938C58B3`〕

**当前落点**：该方法已出现在工作树草稿 `references/storyboard-and-production.md` §“转场选择”，核心 `SKILL.md` 不必重复整张表，只需在“项目拆段/按需读取”加入正确路由和一句决策闸门。

### 2.3 项目任务先做“风险代表镜头”，再批量

**建议规则**：多镜头项目先测试能代表最高风险的少量镜头：环境/尺度、身份一致性、关键交互或特效、情绪或接力结尾。某一高风险效果会跨镜头复用时，先把它单独生成稳定，再加入角色和运镜。

**为什么是核心增量**：现有 `reference-workflows.md` 的“身份 → 动作 → 运镜 → 环境/声音”是单镜头控制层递增；飞书的项目案例增加了横向的“代表镜头闸门”，能防止在风格、角色或天气尚未稳定时批量烧生成次数。〔`feishu-main-text.txt`，§6.4“推荐生成顺序与迭代策略”，行 416–423；§14.3“生成阶段”，尤其“核心镜头测试”“角色一致性验证”“天气效果测试”，行 795–805〕

**当前落点**：工作树草稿 `references/storyboard-and-production.md` 已有“选择核心测试镜头”和“生成顺序”。核心 Skill 只需为长项目路由它，不应再造一份生成顺序。

## 3. 应放入按需 references 的内容

这些内容有价值，但不应常驻 `SKILL.md`。建议优先扩展现有参考文件，只有确实形成独立路由时才新建文件。

### 3.1 扩展 `reference-workflows.md`：视频参考质量与多人隔离

应吸收：

- 参考动作需要主体清晰、无遮挡、光照稳定、动作完整、目标角度接近；时长数字不应固定写死，而应表达为“只保留足以看清一个完整动作节奏的最短片段”。〔主教程 §9.2，行 517–532〕
- 面部表演优先正面或四分之三角度、较小幅度微表情，并把视频限制在眼睑、视线、嘴角、呼吸等可观察层。〔主教程 §9.4，行 544–553〕
- 多角色采用“独立身份参考 → 明确空间槽位 → 顺序化互动”的三层结构；传递物体等接触动作拆成准备、接触、分离。〔主教程 §10.2，行 570–577；§10.4，行 588–602〕
- 人群不逐个定义；让主角清晰，背景人群通过远景、剪影、遮挡或浅景深降复杂度。〔主教程 §10.5，行 603–609〕

不应吸收固定的“3–10 秒”“2–3 秒”平台式经验数值，也不应承诺参考视频一定比文本优先；应把它们写成当前入口待验证的经验，而不是硬规则。

### 3.2 核对并路由现有“镜头意图与转场”草稿

只保留决策索引，不保留术语百科全文。每一项应包含：

`叙事意图 → 首选手段 → 可见画面条件 → 与其他命令的冲突 → 失败时简化方案`

值得收录的高价值映射：

- 推镜：接近重要信息或增强情感；拉镜：揭示环境、抽离或离别；跟拍：展示空间穿越；手持：紧张与纪实；滑动变焦：主体大小近似不变、背景透视变化，用于感知崩塌。〔`feishu-corpus/handbooks/运镜与镜头语言术语手册.txt`，§“摄影机运动”，行 50–109；`feishu-handbook-frame-contents.json` record `C51A6328956690A9C6A007E676894A83`〕
- 情感剪辑点、静止镜头、枕镜、声画对位：用于“何时切”和“为什么停”，而不是装饰性电影术语。〔同手册 §“蒙太奇与叙事节奏”，行 374–397〕
- 纵深构图、镜像、窗框分割：用于在单镜头里表达同时发生的关系、双重性或隔绝。〔同手册 §“空间与深度技法”，行 399–422〕
- 匹配剪辑和声音桥：形成 A 出点/B 入点对，不依赖一句抽象“无缝”。〔主教程 §4.5，行 300–314〕

电影案例只作为检索标签：

- 《公民凯恩》“雪地中的童年”：前/中/背景同时承载不同叙事信息，适合作为深度分层范例。〔`feishu-corpus/movie-cases/公民凯恩 分镜详解.txt`，分镜 3，行 28–38；record `16B544C4D4CAEB7C4B3C35A9E6004FAD`〕
- 《1917》“开场战壕”：运动每次转弯都揭示新空间，适合作为“长镜头必须持续交付信息”的范例。〔`feishu-corpus/movie-cases/1917 分镜详解.txt`，分镜 1，行 6–19；record `34C56E34964D5261C7BF9D225A4279DA`〕
- 《花样年华》“面摊走廊的慢速擦肩”：用空间距离、无接触和无对视表达关系，适合“动作少但关系强”的双人场景。〔`feishu-corpus/movie-cases/花样年华 分镜详解.txt`，分镜 1，行 6–16；record `74A8562E05EC4E17B5B725C8A4C20DD9`〕
- 《疯狂的麦克斯：狂暴之路》总结中的中心构图原则：高速剪辑可把主要视觉信息留在中心，减少观众视线重定位。只吸收原则，不吸收未核验的拍摄比例或技术统计。〔`feishu-corpus/movie-cases/疯狂的麦克斯：狂暴之路 分镜详解.txt`，§“本片核心视觉特征总结”，行 90–101；record `F4763DD3E4E21B48A4CA0677E8AB4731`〕

工作树的 `references/film-grammar.md` 已经采用“视觉问题 → 可提取语法 → 参考案例”的正确形式；`references/storyboard-and-production.md` 已有转场表。建议保留这种分工并从 `SKILL.md` 路由，不要新增第三个重复索引。

### 3.3 核对并路由现有“环境动力与可见反应”草稿

当前 Skill 会写场景与光线，但缺少动态环境的参数化。建议抽象为：

`现象类型 + 强度/密度 + 方向/速度 + 与人物和表面的反应 + 光线如何显形 + 结尾状态`

可吸收的具体方法：

- 雨：强度、雨线方向、湿表面/水洼/飞溅/衣服反应、反光或逆光；
- 雾：层次、可见距离、主体从雾中出现或被吞没的状态；
- 火烟：火源类型、火焰动态、烟的方向、火光对人物与阴影的影响；
- 环绕镜头：方向、半径、速度和主体是否静止四项中，只保留任务真正需要的参数。

来源：主教程 §11.2–11.4，行 621–647；§12.3，行 690–699。这里应提炼通用可见反应，不复制原示例。

工作树的 `references/visual-direction.md` §“天气必须产生环境反应”已经实现这套结构；优先补路由和 eval，而不是继续扩写天气词表。

### 3.4 为长片段/剧情任务增加“场景功能”最小卡片

《剧本提示词专业手册》大量内容与 Seedance 提示词无关，但其中“单场戏五要素”非常适合长片段拆段：目标、障碍、行动、转折、价值变化。它能避免每段只有气氛、没有状态变化。〔`feishu-corpus/handbooks/剧本提示词专业手册.txt`，§3.1“单场戏结构”，行 280–303；`feishu-handbook-frame-contents.json` record `05ACF27E142A9BD8220AE4D9D054998E`〕

建议只在用户要剧情、短片项目或多段叙事时读取，并压缩成三问：

1. 这一段让谁想得到什么？
2. 什么可见事件阻止或改变它？
3. 片段结束后哪一项状态与开头不同？

不要把三幕剧、英雄之旅、救猫咪节拍表等整套编剧框架引入默认 Seedance 工作流。

工作树的 `references/storyboard-and-production.md` §“压缩故事”已经采用了这条最小结构，后续只需验证它不会污染“用户只要一条提示词”的快速输出。

### 3.5 项目模式的视觉连续性板

可吸收“主色、点缀色、镜头间色彩过渡”三项，而不是固定某种废土色盘；天气跨镜头时还要接力方向、密度和光源。〔主教程 §2.4“色彩与光影系统”，行 110–125；§11.1，行 613–620〕

这项只用于多镜头项目。单条快速提示词不需要先做风格圣经或色彩分镜。

工作树的 `references/visual-direction.md` §“视觉圣经/色彩签名”已覆盖该内容；同样需要路由而不是再建文件。

## 4. 建议新增的 eval cases

现有 eval 已覆盖“直接给成品、真实引用、因果对应、最小改动、语义风格词与平台事实分离”。以下 6 个案例能覆盖本次发现的真实缺口。

### 4.1 `reference-video-scope-isolation`

**请求**：`@图片1` 是荒漠女主；`@视频1` 是都市中穿红裙的演员踉跄行走。只迁移疲惫步态和停顿节奏。
**应有**：明确 `@视频1` 只控制步态/节奏；忽略演员身份、红裙、城市、光色；`@图片1` 覆盖身份和服装。
**避免**：把红裙或城市背景带入成品；只写“参考 @视频1”。
**来源**：主教程 §9.3、§9.5，行 533–560。

### 4.2 `match-cut-two-ended-contract`

**请求**：把旋转的银币硬切到夜空圆月，要求匹配剪辑。
**应有**：A 尾帧和 B 首帧分别写清圆形大小、画面位置、运动轴/旋转、明暗或颜色匹配；允许硬切。
**避免**：只写“无缝转场”；把两个镜头混成一个不可执行的连续变形。
**来源**：主教程 §4.5、§12.4；《2001太空漫游》分镜 2。

### 4.3 `two-character-anti-fusion`

**请求**：`@图片1` 的父亲把钥匙交给 `@图片2` 的女儿，8 秒双人镜头。
**应有**：独立身份参考、左右/前后空间槽位、递出→接触→接过的顺序、一次一个关键交互；首轮优先固定或简单镜头。
**避免**：两人同时做多项动作；共享含糊外貌；用“two people exchange a key”带过接触时序。
**来源**：主教程 §10.2、§10.4，行 570–602。

### 4.4 `weather-environment-reaction`

**请求**：雨夜人物奔跑，要求雨真实且连续。
**应有**：雨强/方向/密度、地面或衣物反应、飞溅、光线如何照出雨线、结尾天气状态。
**避免**：只堆“cinematic rain”；同时要求干燥衣服或无反射地面等矛盾状态。
**来源**：主教程 §11.1–11.2，行 613–629。

### 4.5 `risk-first-project-gate`

**请求**：规划 12 镜头短片的生成顺序，包含固定角色、沙尘暴和关键递物。
**应有**：先选少量代表镜头验证环境、身份、关键交互/特效和结尾；沙尘暴单独测试方向与密度；通过后才批量。
**避免**：按 1–12 顺序一次性全部生成；同时调身份、天气、动作和镜头。
**来源**：主教程 §6.4、§14.3，行 416–423、795–805。

### 4.6 `film-principle-not-shot-copy`

**请求**：参考《公民凯恩》的深焦叙事做一个原创家庭场景。
**应有**：提取前/中/后景同时传递关系的原则，换成用户自己的角色、空间和事件。
**避免**：复刻雪地儿童、窗框、签文件等完整标志性调度；长段复制案例提示词。
**来源**：《公民凯恩 分镜详解》分镜 3；主教程版权声明行 923–930；`feishu-storyboard-analysis-text.txt` 版权声明行 38–45。

## 5. 明确排除的内容

### 5.1 排除未经一手官方链接支持的产品参数

不要从飞书教程写入以下硬事实：所谓 Seedance 2.5 官方型号、30/50 秒、4K、50 张参考图、默认 30fps、固定 100–300 英文词、固定无音频能力，以及与 Runway/Kling/Pika 的版本比较。它们集中在主教程行 16、22–36、44–53，没有附可追溯的字节跳动官方链接，且属于高度易变的产品事实。当前 Skill 已有正确闸门：`references/platform-and-versioning.md` §“使用原则”要求以当前界面和官方资料为准，并把课程说法标记为待界面验证。

可保留的只有非数值原则：“参考过多可能增加控制冲突”“实际入口与模型版本要区分”，但仍应由当前界面和官方资料决定具体上限。

### 5.2 排除通用长负面词表和一律禁字/禁 Logo

主教程 §3.4 行 159–166 把通用负面词表称为不可缺少，并列出大量固定禁词；许多电影案例也机械追加 `no text`。这与当前 Skill 的目标不兼容：广告包装、片名和字幕可能本来就是需求，长否定词还会稀释主动作。继续保留现有规则“只处理高概率失败，不附通用禁词清单”。

主教程雨夜示例甚至同时要求“dense neon signs in Chinese, Japanese and Korean characters”和 `no text`（§11.2，行 627–629），内部自相矛盾，不能作为模板。

### 5.3 排除超长角色锚点、每句重复和固定“三图锚定法”

主教程一方面给出很长的角色锚点并要求在每句重复（§3.3，行 146–157），另一方面又承认锚点不是越长越好（行 158）。当前 Skill 已正确改成少量可观察锚点和素材唯一主职责。三视图、面部角度、材质图可以作为高风险角色项目的候选素材，不应变成所有任务的固定上传清单，也不应在用户只要一条短提示词时展示。

### 5.4 排除电影案例中的精确史实、器材和百分比，除非另行核验

这些电影案例是飞书作者的教学整理，不是影片、摄影师或制片方的一手资料。时间点、焦段、快门、摄影机、镜头型号、帧率和“90% 实拍”等数字不能作为确认史实写入 Skill。尤其《花样年华》案例行 8 的“升格至约 6–8fps”表述在术语上可疑；《疯狂的麦克斯》总结行 94 的比例也没有一手出处。可以吸收构图和叙事原则，但删除未核验技术元数据。

### 5.5 排除术语百科、整套编剧体系和整篇案例

- 《运镜与镜头语言术语手册》与《AI 电影镜头提示词万能手册》存在大量重复；只保留任务路由所需的意图索引。
- 《剧本提示词专业手册》的三幕剧、英雄之旅、救猫咪、电视剧结构、类型模板和剧本格式不应进入默认 Seedance Skill。只取“场景需要可见变化”的最小卡片。
- 23 部电影的 138 个分镜不要放进常驻上下文；只建立可检索标签或 6–10 个经验证的模式卡。
- 不复制原文提示词。主教程版权声明明确禁止复制、摘编、翻译、公开发布或用于商业培训（`feishu-main-text.txt` 行 923–930）；两个索引页重复同一限制（`feishu-storyboard-analysis-text.txt` 行 38–45；`feishu-storyboard-library-text.txt` 行 19–26）。Skill 只能重新抽象方法并生成原创任务方案。

### 5.6 排除绝对化构图规则和相互矛盾的速查模板

《AI 电影镜头提示词万能手册》§6.4 行 1242 写“不要居中（除非故意对称）”，但《疯狂的麦克斯》案例又把中心构图视为高速剪辑的核心解决方案（行 90–95）。正确规则不是“不居中”，而是让构图服务于叙事与剪辑负荷。

同样，主教程人群示例行 611 同时写 `deep focus` 和“background softly blurred”，属于光学意图冲突。Skill 应继续执行“一个短镜头一个主镜头意图”和冲突检查，不能照抄示例。

### 5.7 排除工具清单、时间估算和固定交付编码

主教程 §13–14 的 Topaz/RIFE/waifu2x 工具清单、固定 4K/ProRes/H.265 输出建议、10–16 天项目估算属于易变的后期制作建议，不是 Seedance 提示词 Skill 的稳定核心。只有用户明确询问后期流程时才单独核验当前工具和交付规格。
