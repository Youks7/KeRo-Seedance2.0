# 结果诊断与单变量修复

## 先建立因果对应

在分析前记录：

- 原提示词或版本号；
- 对应的视频、截图或用户描述；
- 使用的入口、模型和实际素材；
- 用户认为成功与失败的具体片段。

如果存在多条提示词或多段视频但对应关系不清，只询问对应关系。不得凭文件顺序或印象猜测。

## 定位主要失败层

| 症状 | 常见主要原因 | 第一轮只改 |
|---|---|---|
| 主体或角色漂移 | 身份基准不唯一、锚点过多或遮挡过强 | 统一身份基准，保留少量可见锚点 |
| 肢体融合或动作断裂 | 同时动作太多、接触阶段未拆开 | 减少动作，写准备→接触→分离 |
| 镜头乱推拉或突然切换 | 多个镜头命令冲突 | 只保留一个主运动和明确终点 |
| 时间轴拥挤 | 镜头数和动作量超过时长 | 删镜头或删次要动作 |
| 风格变弱 | 关键风格锚点被大量技术说明稀释 | 恢复少量强风格词，删除装饰参数 |
| 产品结构变化 | 不可变结构不清或特效过多 | 固定轮廓、接口和标签，只留一种特效 |
| 转场跳变 | 上一段出点没有在下一段复述 | 明写位置、姿态、镜头速度和光向 |
| 文字不可读 | 字数过多、画面运动或透视变化 | 缩短文字并留在稳定尾镜，必要时后期合成 |
| 音画不同步 | 对白过长或同步节点不清 | 缩短对白，只标一个同步动作 |

## 最小修改法

每轮只选择一个层：

1. 身份或产品结构；
2. 动作因果；
3. 镜头；
4. 时间分配；
5. 光色与风格；
6. 声音；
7. 装饰细节。

输出新版完整提示词，不只给零散补丁。随后写明下一轮只观察哪一个变量。

## 不要过度工程化

以下内容经常让结果变差：

- 重复十几次同一身份要求；
- 发明无法从参考图确认的精确纹理拓扑；
- 用长篇物理解释替代一个清楚动作；
- 同时加入大量否定词；
- 为每个镜头堆焦段、光圈、快门、帧率、色彩空间和多个运镜；
- 在一次重试中同时重写主体、动作、镜头、风格和声音。

如果用户的实际结果证明某个简洁风格标签或结构有效，保留它。不要让通用清单覆盖直接证据。

## 评审输出

使用以下顺序：

```text
主要问题：具体到时间点或画面。
证据：结果中可观察到什么。
最可能原因：对应提示词中的哪一层。
新版完整提示词：只改变关键层。
下一轮观察：一个变量。
```
