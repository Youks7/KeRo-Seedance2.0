# 平台能力与版本处理

## 使用原则

把 Seedance、即梦网页端、豆包入口和火山方舟 API 视为不同产品表面。不要假设它们在同一天拥有相同模型名称、时长、分辨率、素材格式或计费方式。

遇到版本号时：

1. 记录用户实际选择的模型名称和入口。
2. 以当前界面显示的参数为准。
3. 需要外部核实时，只使用字节跳动 Seed、即梦或火山引擎官方资料。
4. 把无法官方确认的课程说法标记为“待界面验证”。

截至 2026-07-12，字节跳动 Seed 官方公开材料可确认 Seedance 2.0；未检索到官方 Seedance 2.5 模型页。用户资料中的“2.5”可能是产品入口、课程命名、灰度版本或非公开更新，不得自动当成官方统一型号。

## 已有官方依据

官方 Seedance 2.0 发布材料说明：

- 支持文本、图片、视频和音频输入；
- 可组合参考最多 9 张图片、3 段视频和 3 段音频；
- 支持最长 15 秒的高质量多镜头音视频输出；
- 支持视频延长、视频编辑和多模态参考；
- 真人人像参考需要本人验证或事先取得合法授权。

这些数据只用于 2.0 的已验证基线。生成前若界面不同，以当前界面为准。

## 素材引用规范

中文界面优先使用：

- 图片：`@图片1`、`@图片2`；
- 视频：`@视频1`、`@视频2`；
- 音频：`@音频1`、`@音频2`。

遵守以下约束：

- 编号必须对应真实上传顺序；
- 每个素材至少声明一个明确用途；
- 不使用 `@img1`、`图片1`、`@图1` 等混合写法；
- 界面自动插入的标签与本规范不同时，保留界面原标签；
- 不把未上传素材写进最终提示词。

## 入口选择

按实际界面名称选择，但保持以下逻辑：

- 纯文本：适合开放创意和快速概念验证；
- 首帧或首尾帧：控制开始或结束构图；
- 全能参考：组合角色、场景、动作、运镜、节奏与音色；
- 视频延长：把生成时长理解为新增段长度，并明确从原视频哪一状态继续；
- 视频编辑：明确保留项、替换项和不得改变项。

## 规格表达

分开描述：

- 平台参数：用户界面真实可选的时长、比例、分辨率和帧率；
- 画面风格：电影感、颗粒、动态范围、镜头特征和调色倾向。

不要写“8K电影感”来暗示平台会输出 8K。若用户要求高分辨率，输出“选择当前入口可用的最高分辨率，并在后期按需超分”。

## 易变信息检查表

以下问题必须核实或注明未核实：

- 当前模型名称与可用地区；
- 单次生成时长；
- 输入文件数量、大小、格式和视频总时长；
- 输出分辨率与画幅；
- 是否原生生成声音；
- 人像验证与内容合规规则；
- API 与网页端是否拥有同一能力；
- 价格、积分或 Token 消耗。
