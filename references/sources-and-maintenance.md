# 来源与维护

## 来源层级

1. 当前产品界面与官方帮助；
2. 字节跳动 Seed 官方模型页和发布稿；
3. 火山引擎官方产品或 API 文档；
4. 用户提供的课程、案例和经验；
5. 社区测试与个人经验。

低层级来源不得覆盖高层级的明确事实。经验性结论应标注适用版本和测试条件。

## 更新规则

平台发生变化时：

1. 更新 `platform-and-versioning.md` 的核实日期；
2. 保存官方链接；
3. 区分即梦、豆包和 API；
4. 删除失效的硬限制；
5. 用当前界面复测至少一个纯文本、一个图片参考和一个多模态任务；
6. 重新运行 Skill 校验。

不要把课程中的“Seedance 2.5”直接改写成官方型号，除非出现可核实的官方发布或用户当前界面证据。
