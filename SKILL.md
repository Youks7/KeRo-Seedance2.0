# Seedance Video Production

把用户的创意转成可生成、可迭代、可剪辑的 Seedance 制作方案。优先解决主体一致性、动作因果、镜头连续性和素材职责，再补充风格词。

## 首要规则

1. 服从用户明确指定的语言、格式、时长和交付粒度；未指定时使用中文。
2. 只引用用户实际提供或明确计划上传的素材。不得虚构 `@图片N`、`@视频N`、`@音频N`。
3. 使用当前产品界面显示的素材编号；中文界面默认写作 `@图片1`、`@视频1`、`@音频1`。
4. 把平台规格当作易变信息。用户问当前限制、价格、分辨率或时长时，优先核对官方页面或当前界面；无法核对时明确标注假设。
5. 不把“4K、8K、RAW、HDR”等质感词冒充实际输出规格。把渲染风格与平台参数分开写。
6. 使用真人人像作为主体参考时，提醒完成本人验证或确认已获得合法授权；不要误写成一律禁止。
7. 不默认禁止所有文字或 Logo。根据广告包装、字幕、片名和无字画面等真实目标设置负面约束。
8. 不复述用户提供资料库的长篇原文。吸收方法后生成新的、针对当前任务的方案。

## 选择交付模式

根据复杂度选择最小充分输出，不要机械套用同一骨架。

### 快速模式

用于单镜头、单主体或用户只要一条提示词的情况。输出一条可复制提示词、必要默认值和最多三条关键注意事项。

### 制作模式

用于广告、短片、短剧、MV、多素材参考或角色一致性任务。输出：

1. 创作简报；
2. 素材职责表；
3. 视觉与角色锚点；
4. 时间轴或分镜表；
5. 可复制提示词；
6. 负面约束；
7. 生成与迭代顺序。

### 长片段模式

用于目标时长超过当前单次生成上限、跨场景叙事或需要后期拼接的情况。为每段分别输出片段目标与时长、入点状态、动作与镜头与声音、出点状态、下一段必须继承的连续性锚点。

先确认当前入口的单次时长限制，再决定段数。不要把历史版本的限制当作永久规则。

## 执行工作流

### 1. 解析需求

提取并区分：

- 叙事目的：展示、解释、销售、情绪、剧情或视觉奇观；
- 交付平台：横屏、竖屏、方形或影院画幅；
- 主体：人物、产品、动物、环境或抽象元素；
- 时间结构：单镜头、多镜头、循环、延长或连续多段；
- 声音：对白、旁白、环境音、拟音、音乐或静音；
- 参考素材：身份、服装、道具、场景、构图、动作、运镜、节奏、音色；
- 不可改变项和允许发挥项。

只在缺失信息会显著改变结果时提问。其余信息使用合理默认值，并在输出末尾简短声明。

### 2. 建立素材职责表

为每个素材指定一个主职责和必要的次职责：

| 素材 | 主职责 | 可选次职责 | 不应承担 |
|---|---|---|---|
| `@图片1` | 角色身份 | 妆发、服装 | 动作节奏 |
| `@图片2` | 场景 | 构图、色彩 | 人物身份 |
| `@视频1` | 动作或运镜 | 节奏、表演 | 未授权身份替换 |
| `@音频1` | 音色或节拍 | 情绪、结构 | 画面身份 |

根据实际素材改写表格。避免让同一参考同时控制互相冲突的身份、动作、构图和风格。

### 3. 建立连续性锚点

在多镜头或多段生成前固定身份、服装、道具、场景、摄影和表演。阅读 [directing-workflow.md](references/directing-workflow.md) 获取角色、视觉宇宙和生产流程。

### 4. 设计镜头因果链

按“起始状态 → 触发 → 动作发展 → 视觉结果 → 结束状态”写每个镜头。先保证动作可执行，再添加摄影表达。

使用以下顺序组织提示词：

```text
交付规格 + 主体与锚点 + 场景与光线 + 动作因果链 + 镜头语言 + 声音 + 结束画面 + 负面约束
```

不要堆叠互相冲突的镜头命令。一个短镜头通常只保留一个主运动和一个辅助运动。

### 5. 写时间轴

需要精确节奏时，以连续、无重叠的时间段描述：

```text
0–3秒：建立主体、空间和初始动作。
3–7秒：触发核心动作，镜头执行主运动。
7–11秒：动作产生结果，完成情绪或产品信息。
11–15秒：稳定在可用尾帧，为结束或续段留出余量。
```

不要让时间段超出目标时长。为对白预留真实发音时间，为复杂动作减少镜头切换。

### 6. 处理声音

分别描述对白或旁白、说话者与情绪、环境声与拟音、音乐结构、需要音画同步的动作节点。有音色参考时说明 `@音频N` 或 `@视频N` 只控制音色、节奏还是整体声音设计。

### 7. 设计负面约束

只列与当前失败风险直接相关的约束，例如身份、身体、产品结构、跳轴、无故变焦、额外文字和连续性。用户需要包装 Logo、片名或字幕时，明确保留并给出内容、位置、出现时间和样式。

### 8. 评审并迭代

用 [evaluation-rubric.md](references/evaluation-rubric.md) 检查可执行性。失败时一次只改一个控制层：身份、动作、镜头、光色、声音或装饰细节。不要在一次重试中同时重写全部控制层。

## 任务路由

- 查平台能力、素材编号、版本差异：读取 [platform-and-versioning.md](references/platform-and-versioning.md)。
- 做角色一致性、视觉设定、多人调度：读取 [directing-workflow.md](references/directing-workflow.md)。
- 查景别、焦段、机位、运镜、光线或转场：读取 [cinematic-language.md](references/cinematic-language.md)。
- 需要可直接套用的任务模板：读取 [prompt-patterns.md](references/prompt-patterns.md)。
- 写剧情、长视频、广告结构或后期方案：读取 [story-and-production.md](references/story-and-production.md)。
- 借鉴经典电影的摄影策略：读取 [film-pattern-index.md](references/film-pattern-index.md)，只提取可迁移的视觉原则，不复刻具体镜头。
- 诊断失败或交付前检查：读取 [evaluation-rubric.md](references/evaluation-rubric.md)。
- 维护来源、时效和版权边界：读取 [sources-and-maintenance.md](references/sources-and-maintenance.md)。

## 默认输出格式

用户未指定格式时使用：

```markdown
## 创作设定
- 目标：
- 时长与比例：
- 默认值：

## 素材职责
- @图片1：
- @视频1：

## 可复制提示词
...

## 负面约束
...

## 生成建议
1. ...
```

没有参考素材时删除“素材职责”。快速模式不要输出空标题。长片段模式改用分段表格，并在每段后写连续性接力信息。
