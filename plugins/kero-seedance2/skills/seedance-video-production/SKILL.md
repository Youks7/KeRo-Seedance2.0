# Seedance 视频制作

把创意或已有结果转成短、强、可复制、可迭代的 Seedance 提示词。默认先交付成品，不先讲方法。

## 工作原则

1. 分开处理两类证据。判断提示词效果时：已确认对应关系的实际生成结果 > 用户明确反馈与素材 > 通用经验。回答平台型号、时长、分辨率、价格或地区可用性时：当前产品界面 > 官方资料 > 用户课程与二手材料 > 通用经验。不要让其中一条证据链覆盖另一条。
2. 评审已有结果前，先确认哪条提示词生成了哪段视频。因果对应不清时只问这一项，不猜。
3. 只引用真实提供或明确准备上传的素材，不虚构 `@图片N`、`@视频N`、`@音频N`。
4. 使用真人人像作为主体参考时，确认当前入口要求的本人验证或用户已取得合法授权。用户未说明时，在成品后只加一行确认提示；不要把这条入口要求写成一律禁止真人参考。
5. 使用参考视频时建立“采用 / 忽略 / 覆盖”合同：采用哪一层动作、表演、运镜或节奏；忽略哪些身份、服装、场景、光色或声音；冲突时由哪张图片或哪条文本要求覆盖。只写“参考@视频1”不合格。
6. 保留用户明确要求或实际结果已验证有效的语义引导锚点。`国家地理`、`IMAX`、`4K`、`240fps`、`升格慢动作` 等词可以描述期望的画质、摄影和运动观感；`8K` 等其他锚点只在用户要求或当前任务已验证有效时使用。不要因为它们不等于导出参数就删除，也不要据此断言模型内部启用了特定训练模块。
7. 把提示词语义与产品参数分开处理。提示词可以写语义引导锚点；只有用户询问实际导出分辨率、帧率、时长、输入数量、价格或地区可用性时，才读取 [platform-and-versioning.md](references/platform-and-versioning.md) 并核对当前官方资料或界面。
8. 不用规则数量冒充控制力。优先写可见动作、清楚镜头和稳定结果，删除无法被画面验证的装饰性约束。

## 选择任务路径

只选满足需求的最短路径：

- **直接生成**：用户给创意并要提示词。直接输出一条成品提示词。
- **视觉开发**：用户要风格、镜头、光色或天气方案。先建立简洁视觉圣经，再转成镜头可观察属性。
- **多素材制作**：用户提供图片、视频或音频。先在内部确定每个素材的唯一主职责，再输出成品；复杂映射才展示素材表。
- **延长或编辑**：先写必须继承和必须改变的状态，再给操作提示词。
- **结果诊断**：先确认提示词与结果对应，再定位一个主要失败层，只改最小必要部分。
- **分镜与项目制作**：先压缩故事功能和视觉变化，再设计分镜、转场、核心测试镜头、生成顺序与后期闭环。
- **项目拆段**：目标超过当前单次能力或跨多个场景时，按可稳定接力的出点拆段。

## 编译成品提示词

### 1. 提取硬约束

识别目标、时长、画幅、主体、场景、关键动作、声音、参考素材、不可改变项和结尾。仅当缺失信息会明显改变方案时提问；其余使用合理默认值。

### 2. 控制复杂度

- 每个短镜头只承担一个主动作和一个主镜头意图。
- 10 秒任务通常控制在 1–3 个镜头；动作越复杂，镜头越少。
- 每个主体只保留少量可观察锚点，不发明精确纹理拓扑或冗长生物力学说明。
- 多素材任务给每个素材一个主职责，避免一个参考同时控制身份、动作、构图和风格。
- 负面约束只处理高概率失败，不附带通用禁词清单。

### 3. 使用可见因果

按以下顺序组织，但不要机械显示字段名：

```text
语义与风格锚点 → 画幅与时长 → 场景与主体 → 分时段动作 → 每段镜头 → 声音 → 结束画面 → 必要限制
```

每个动作写成“起始状态 → 触发 → 动作 → 可见结果”。每个镜头写清景别或焦段、一个主运动和决定性画面。精确节奏才使用连续且不重叠的时间码。

相邻镜头先判断是连续接力还是有意剪辑：连续接力要复述出点状态；有意剪辑要指定动作、形状、颜色、构图或声音中的一个匹配维度，并分别写清 A 尾帧与 B 首帧。不要只写“无缝转场”。

### 4. 静默验收

交付前内部检查：

- 是否能一眼看出主体在做什么；
- 时间码是否连续且不超时；
- 镜头命令是否互相冲突；
- 所有 `@引用` 是否真实存在且编号一致；
- 真人人像任务是否已确认当前入口要求的本人验证或合法授权；
- 是否有能稳定生成或接力的结束状态；
- 是否把语义锚点误写成真实平台规格或模型内部机制；
- 是否添加了用户没要求的表格、说明或限制。

未通过就先修正，不展示评分过程。

## 输出契约

### 用户只要提示词

第一行直接开始可复制提示词。默认不输出创作简报、素材职责表、评分表、负面词解释或长篇教程。只有存在关键默认值或未确认的真人人像入口要求时，提示词后用一行说明。

### 用户要制作方案

根据用户要求选择最小充分内容，不机械套全套。完整项目可按顺序输出：

1. 创意目标与视觉圣经；
2. 分镜、节奏与衔接；
3. 每镜或每段可复制提示词；
4. 必要的素材职责与连续性接力；
5. 核心测试镜头、生成顺序和后期要点。

用户只要提示词时仍然遵守“成品优先”，不要输出上述制作框架。

### 用户要评审或修复

按顺序输出：

1. 主要问题及直接证据；
2. 最可能原因；
3. 只改关键层的新版完整提示词；
4. 下一轮只观察的变量。

不得在因果对应未确认时断言某条提示词导致某个结果。

## 按需读取

- 需要文生视频、参考生成、广告、延长、编辑或多段模板时，读取 [prompt-recipes.md](references/prompt-recipes.md)。
- 需要建立视觉宇宙、选择镜头、设计光色、天气或复杂运镜时，读取 [visual-direction.md](references/visual-direction.md)。
- 需要分配素材职责、建立参考库、迁移表演、保持角色或产品一致、处理多角色和跨段连续性时，读取 [reference-workflows.md](references/reference-workflows.md)。
- 需要剧本压缩、分镜表、节奏、转场、长项目生产或后期闭环时，读取 [storyboard-and-production.md](references/storyboard-and-production.md)。
- 用户提到电影、导演或摄影师作为视觉参考时，必须读取 [film-grammar.md](references/film-grammar.md)，把参考转换成可观察视觉语法，不只输出名字。
- 需要分析失败结果或设计单变量重试时，读取 [troubleshooting.md](references/troubleshooting.md)。
- 需要当前平台规格、官方依据或版本处理时，读取 [platform-and-versioning.md](references/platform-and-versioning.md)。

不要为了显得完整而读取所有参考文件。
