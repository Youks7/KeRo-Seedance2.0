# 提示词配方

只读取与当前任务匹配的配方。替换占位内容并删除无关字段，不要把方括号原样交付给用户。

## 纯文本与多镜头

优先使用以下紧凑结构：

```text
【风格】[少量强风格锚点]
【时长与比例】[时长]，[画幅]
【场景】[可观察环境、时间、天气和光线]

[时间段] 镜头1：[短标题]（[景别/焦段/主运动]）
[一个主动作、决定性细节和可见结果]

[时间段] 镜头2：[短标题]（[景别/焦段/主运动]）
[一个主动作、决定性细节和可见结果]

【声音】[环境声、拟音、对白或音乐；不需要就删除]
```

10 秒通常使用 1–3 个镜头。镜头数量不是质量指标；复杂动作优先减少切换。

## 单镜头

```text
[时长、比例、风格]。[主体]在[场景]从[起始状态]开始；因为[可见触发]，[连续动作]，最终[稳定结果]。摄影机以[景别和角度]开始，只执行[主运动]；[光线与声音]。[一至两项必要限制]。
```

适合单一动作、情绪瞬间、产品英雄镜头和用户明确要求“一条提示词”的任务。

## 多素材参考

```text
以@图片1作为[身份/产品结构/场景]的唯一基准，只保留[可观察锚点]；@图片2只控制[服装/环境/构图]；只参考@视频1的[动作节奏/镜头轨迹/剪辑节奏]，不继承其中的[身份/服装/场景]。

[目标时长与画幅]。[分时段动作和镜头]。最终停在[稳定尾帧]。保持[少量不可变项]。
```

最终提示词只能出现用户实际提供的编号。素材职责简单时不要额外输出表格。

## 产品广告

```text
[时长、比例、调性]产品广告。@图片1是产品外轮廓、比例、材质与包装的唯一基准。[时间段1]以[英雄角度]建立产品；[时间段2]通过[动作或环境反应]展示一个功能证据；[时间段3]回到稳定产品定格，[保留或后期添加的品牌信息]。灯光[主光和背景]，摄影机只执行[主运动]。不得改变[结构、接口、标签位置]。
```

不要同时要求展示包装文字和禁止全部文字。关键小字优先后期合成。

## 视频延长

```text
向后延长@视频1新增[X]秒。严格继承原视频最后一帧的[人物位置、姿态、道具状态、镜头方向与速度、光线、环境声]。随后[一个动作发展]；摄影机继续[原有运动或自然停稳]，不得重置机位和主体姿态。结尾稳定在[下一段可接力状态]。
```

向前延长时写首帧之前的状态，并让新增段结尾准确进入原首帧。

## 视频编辑

```text
编辑@视频1。完整保留[镜头、动作、节奏、场景、声音]；只把[目标元素]替换为@图片1中的[对象]。替换后继承原有比例、遮挡、光向、接触关系和动作时序；其他元素不变。
```

一次只编辑一个控制层。若用户同时要求换角色、换场景、改动作和改镜头，先拆成多轮。

## 音乐卡点

```text
画面使用[真实素材范围]，@音频1只控制[节拍/情绪/音色]。在[已知时间点]让[动作峰值或构图变化]与声音同步；高潮发生在[时间点]，结尾为[稳定画面]。保持[身份、产品或色彩锚点]一致。
```

没有实际听取或分析音频时，不虚构 BPM、鼓点或精确时间点。

## 多段项目

每段分别写：

```text
片段N：[时长]
入点：[继承上一段的可见状态]
本段唯一目标：[事件]
动作：[起点 → 触发 → 动作 → 结果]
摄影：[景别、角度、一个主运动]
声音：[连续环境、对白或音乐节点]
出点：[稳定尾帧]
接力：[下一段第一句必须复述的状态]
```

优先在动作完成、镜头停稳、遮挡、黑场、失焦或声音段落结束处拆段。
