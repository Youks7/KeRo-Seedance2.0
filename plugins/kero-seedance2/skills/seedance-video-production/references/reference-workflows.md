# 参考素材、表演与连续性

在任务包含图片、视频、音频、多角色、产品一致性、表演迁移、延长或编辑时读取。

## 目录

1. 建立参考素材库
2. 分配素材职责
3. 建立主体锚点
4. 迁移动作与表演
5. 调度多角色与人群
6. 跨镜头接力与编辑
7. 生成顺序

## 建立参考素材库

大型项目可把素材分成四组：

```text
01_主体身份/全身_正面_四分之三_侧面
02_场景气氛/全景_中景_材质_光线
03_服装道具/结构_接口_标签_使用状态
04_镜头参考/构图_动作_运镜_节奏
```

这是一种生产组织方法，不代表每次都要上传全部素材。只选择当前镜头真正需要的参考，素材越多越要明确职责和优先级。

## 分配素材职责

先为每个素材指定一个主职责：

| 素材类型 | 常用主职责 | 不应默认承担 |
|---|---|---|
| 人物图片 | 身份、脸部、发型 | 动作节奏、镜头轨迹 |
| 服装或产品图片 | 结构、材质、标签 | 人物身份、环境运动 |
| 场景图片 | 空间、构图、光色 | 角色身份 |
| 参考视频 | 身体动作、微表情、运镜、节奏之一 | 未声明的人物身份、服装与场景 |
| 参考音频 | 音色、节拍、情绪、声音结构之一 | 画面身份与构图 |

只有素材关系复杂或用户明确要求时才展示职责表。简单任务直接把职责写进成品提示词。

冲突时使用以下优先级：用户明确指定 → 身份与产品结构 → 动作因果 → 构图与镜头 → 风格装饰。不要让风格参考覆盖主体身份。

## 建立主体锚点

### 人物

身份锚点选择稳定且可见的特征：脸型、发型、体型、服装轮廓、显著配饰、必要道具。区分：

1. 永久身份；
2. 当前场景服装；
3. 当前情绪与表演；
4. 当前镜头动作。

不要把临时情绪写进永久身份，也不要发明精确斑纹坐标、每根毛发走向或无法从素材确认的身体参数。

使用真人人像作为主体参考时，确认当前入口要求的本人验证或已取得的合法授权；不要把这条平台要求误写成一律禁止真人参考。

### 产品

优先固定：

- 外轮廓和比例；
- 接口、按钮、开合结构；
- 材质与主要反射；
- 标签和 Logo 位置；
- 允许变化与禁止变化的部位。

### 材质

把“真实”具体化为磨损、褶皱、灰尘、氧化、指纹、湿润、织物纤维或反射方式。只选择与镜头距离能看见的细节。

## 迁移动作与表演

文本说明“做什么”，参考视频更适合说明“怎么做”。使用前检查：

- 主体清楚，动作没有被遮挡；
- 光线稳定，没有强烈闪烁；
- 动作完整且只包含一个主要节奏；
- 拍摄角度接近目标镜头；
- 表演强度与项目气质匹配。

只保留足以完整看清一次主要动作节奏的最短片段；具体时长以素材动作和当前入口限制为准，不套用固定秒数。

### 表演迁移公式

```text
以@图片1为身份基准。只参考@视频1的[身体动作/步态/停顿/眼口微动作/镜头轨迹]，
把该节奏迁移到当前角色；不参考原视频的人物身份、服装、场景、光色和声音。
[说明必须改变的方向、速度或幅度]。保持[身份、产品或场景锚点]。
```

发生冲突时明确写：参考视频保留哪一层，文本覆盖哪一层。

### 微表情

用可见变化代替情绪标签：眼睑开合、视线落点、瞳孔变化、嘴角、下颌、吞咽、呼吸和肩部张力。优先正面或四分之三角度、小幅变化和短片段。

## 调度多角色与人群

### 多角色三层控制

1. **身份隔离**：每个角色独立参考图、名称和明显不同的轮廓或服装色；
2. **空间隔离**：明确左/右、前/中/后景、朝向、距离和视线；
3. **动作隔离**：明确谁先动作、谁后反应，把接触拆成准备、接触、分离。

示例：

```text
角色A位于左前景，先后退一步；角色B位于右侧中景，看到A后才抬头。
两人的动作不同时发生，背景人物不穿过二人之间。
```

先测试固定镜头和简单站位，再加互动和镜头运动。必要时用背影、剪影、浅景深或反应镜头减少同时显示的清晰面孔。

### 人群

背景人群用整体描述：移动方向、密度、服装范围、共同光源和氛围。不要逐个定义身份。用远景、剪影或适度虚化降低细节竞争，主角保持清晰。

## 跨镜头接力与编辑

记录并复述：

| 类别 | 必须接力的状态 |
|---|---|
| 人物 | 位置、朝向、视线、姿态、情绪强度 |
| 道具 | 所在手、开合、损伤、相对位置 |
| 摄影 | 景别、轴线、运动方向、速度、机位高度 |
| 环境 | 光向、天气、背景运动、烟尘或水面状态 |
| 声音 | 环境声、音乐小节、对白尾音 |

下一段第一句直接复述上一段出点，不写“接着上一段”这种不可见指令。

编辑任务先列：完整保留、只允许改变、变化后必须继承。一次只编辑一个控制层；修复区域或对象越小，越容易保持其他内容不变。

## 生成顺序

1. 验证身份、产品或场景的静态结构；
2. 验证一个主动作或微表情；
3. 加入一个主镜头运动；
4. 加入天气、次要角色和声音；
5. 锁定可用尾帧后生成续段。

如果用户只要一条提示词，不展示这套流程；直接用它压缩成品。
